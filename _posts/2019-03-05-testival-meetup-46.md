Regular talks (10-30 minutes each)

#1 Marko Kruljac (Bornfight): Continuous code quality testing

"Today, I will create unmaintainable legacy code" - said no one ever, but it still happens every day. When unhealthy code becomes a part of the production, it gets very difficult (expensive) to correct the mistakes, and more often than not, that bad code doesn't get retroactively fixed. This leads to frustration, bugs and, in the end, very high financial expenses. What if we could automatically detect unmaintainable and unhealthy code as soon as it gets created - and stop it before it's too late. The solution might be in the continuous testing approach and static code analysis.
