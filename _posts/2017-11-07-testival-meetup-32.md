Regular talks (10-30 minutes)

#1 Marko Kruljac: Integrating Jenkins with your GitHub Pull Request Workflow

Recently we decided to switch to a Pull Request Workflow on GitHub. This means that almost every change to the code-base requires someone to review and approve the change before the code gets integrated. Sometimes, bad code gets queued for a review. You can detect bad code often just by looking at it or running tests and seeing that some have failed. Jenkins is a CI system that allows you to automatically detect code that is clearly bad, or code that does not pass all automated tests - without your engineers wasting even a minute. We will show how to setup an example project and integrate it with GitHub Pull Requests.
